LedgerRocket is a programmable double-entry bookkeeping platform that transforms financial events into balanced transfers using template-driven business logic.

## Key Capabilities

- **Multi-tenant isolation** — each site is a fully independent tenant boundary
- **Dimensional account model** — accounts classified by EntityScope, CounterpartyType, FunctionalDomain, and ValuationRole
- **Template-driven processing** — JSON templates with CEL expressions define how events become transfers
- **Maker-checker governance** — templates go through DRAFT → PENDING_APPROVAL → LIVE → DEPRECATED lifecycle
- **Real-time balances** — TigerBeetle-backed balance queries with sub-millisecond latency
- **Atomic event processing** — events produce balanced double-entry transfers or fail entirely

## Architecture

LedgerRocket consists of four microservices:

| Service | Role |
|---------|------|
| **Ledger Service** | Reference data authority — sites, entities, ledgers, accounts, account codes |
| **Accounting Rule Engine** | Template lifecycle — authoring, validation, maker-checker approval |
| **Event Service** | Atomic event processing — validates events, evaluates templates, creates transfers |
| **Balance Service** | Balance queries — real-time (TigerBeetle), historical (Athena), chain analysis |
